# MVP Setup — Google Drive + MCP

End-to-end setup for the first vertical slice of Rawgrowth AIOS: **connect Google Drive → data ingested to local SQLite → query it from Claude Desktop (or any MCP client) using your Claude subscription**.

---

## 1. Google Cloud Console

You've already done this, but for reference:

1. **APIs & Services → Library** → enable **Google Drive API** and **People API** (for user info).
2. **OAuth consent screen**:
   - User type: External
   - App name: Rawgrowth AIOS
   - Scopes: `.../auth/drive.readonly`, `.../auth/userinfo.email`
   - Test users: add your email while the app is in Testing mode.
3. **Credentials → Create Credentials → OAuth client ID**:
   - Application type: Web application
   - Authorized redirect URI: `http://localhost:3000/api/integrations/google-drive/callback`
4. Copy the Client ID and Client Secret into `.env`.

For production, you'll need to submit the app for Google verification since `drive.readonly` is a "sensitive" scope.

---

## 2. Local env

`.env` at repo root:

```
GOOGLE_CLIENT_ID="…"
GOOGLE_CLIENT_SECRET="…"
GOOGLE_OAUTH_REDIRECT_URI="http://localhost:3000/api/integrations/google-drive/callback"
```

Keep this file out of git — `.env*` is already in `.gitignore`.

---

## 3. Connect Google Drive

```
npm run dev
```

1. Open http://localhost:3000/integrations
2. Click **Connect** on the Google Drive card.
3. Click **Connect with Google Drive** in the slide-over.
4. You'll be redirected to Google, approve the scopes, and land back on `/integrations?connected=google-drive&synced=N`.

On first connection the callback runs an initial sync (up to 50 most-recently-modified files), stored at `.data/aios.db`:

- `bronze_drive_files` — raw payloads, idempotent on file id.
- `drive_files` — normalised metadata (title, mime, owner, modified timestamp).
- `drive_file_chunks` — ~3200-char text chunks for Google Docs + plain-text files.
- `drive_files_fts` — FTS5 index over title + chunk text for keyword search.

Re-sync any time:

```
npm run drive:sync           # default 100 files
npm run drive:sync -- 500    # or a custom cap
```

## 4. MCP server — plug into Claude Desktop / Claude Code / Cursor

The MCP server lives at [scripts/mcp-server.ts](scripts/mcp-server.ts) and runs over stdio so any MCP-compatible client can spawn it.

### Quick sanity check

```
npm run mcp
```

Should print `[rawgrowth-aios MCP] ready on stdio` and then wait for JSON-RPC on stdin. Ctrl-C to exit.

### Claude Desktop

Edit `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "rawgrowth-aios": {
      "command": "npx",
      "args": [
        "tsx",
        "/Users/jamesoldham/Desktop/Coding/rawgrowth/rawgrowth-aios/scripts/mcp-server.ts"
      ]
    }
  }
}
```

Restart Claude Desktop. In a new chat, you should see the four Rawgrowth tools available in the tool picker.

### Claude Code

```
claude mcp add rawgrowth-aios npx tsx /Users/jamesoldham/Desktop/Coding/rawgrowth/rawgrowth-aios/scripts/mcp-server.ts
```

### Cursor

Settings → MCP → add the same command/args as Claude Desktop's config.

## 5. Try it

Once connected in Claude Desktop, ask:

- *"Using rawgrowth-aios, list my 10 most recently modified Google Drive files."* → calls `list_recent_drive_files`.
- *"Search my Drive for anything about pricing."* → calls `search_drive("pricing")`.
- *"Pull up the Q4 planning doc and summarise it."* → `search_drive` then `get_drive_file`.
- *"Is my Drive index up to date?"* → `drive_sync_status`.

Every response is generated by Claude using **your Claude subscription** — Rawgrowth just serves the data over MCP. That's the unit-economics unlock for the product.

## Tools exposed

| Tool | Input | Returns |
| --- | --- | --- |
| `search_drive` | `query`, optional `limit` (max 30) | FTS5 snippets with file_id + title + link |
| `get_drive_file` | `file_id` | Full markdown-formatted content + metadata |
| `list_recent_drive_files` | optional `limit` (max 100) | Most-recently-modified files |
| `drive_sync_status` | — | Last sync time + total file count |

## Known gaps (explicit Phase-1 deferrals)

- **Single-tenant**: no `company_id` columns yet. When you go multi-tenant, every SQLite read needs that filter.
- **Keyword search only**: FTS5 beats naive LIKE but isn't semantic. Add `text-embedding-3-large` + pgvector when you port to Postgres.
- **Tokens stored in plaintext** in SQLite. Wrap with envelope encryption (AWS KMS / Vercel encrypted env) before prod.
- **Text-like mime types only** index content (Google Docs, plain text, markdown, CSV, JSON). PDFs, Sheets, Slides skipped for MVP — add extractors as needed.
- **No incremental sync**: each sync pulls the most recent 100 files fresh. For prod use Drive's `changes.list` cursor saved in `sync_state.cursor`.
- **stdio transport only**: fine for local Claude Desktop. For remote multi-tenant, expose over HTTP/SSE and authenticate with per-tenant bearer tokens.
